Linux下MySQL和Java的配置

1. Java的配置

1.1 查看Java环境

  • 查看安装的 Linux 系统中是否已经安装了 Java，输入命令

    java -version

    如果已经存在了 Java 的环境如下图，那么执行下面命令，如果不存在请直接跳转到下载配置

  • 输入命令：

    rpm -qa | grep java

    以上文件中，noarch 文件不用删除。下面执行删除命令：

    rpm -e --nodeps java-1.7.0-openjdk-1.7.0.221-2.6.18.0.el7_6.x86_64
rpm -e --nodeps java-1.7.0-openjdk-headless-1.7.0.221-2.6.18.0.el7_6.x86_64
rpm -e --nodeps java-1.8.0-openjdk-1.8.0.212.b04-0.el7_6.x86_64
rpm -e --nodeps java-1.8.0-openjdk-headless-1.8.0.212.b04-0.el7_6.x86_64

    No such file or directory 表示删除成功

1.2 安装Java

  • 下载安装包

    去 oracle 官网下载 Linux 版本压缩包


    下载地址：https://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html

  • 解压

    键入命令：ls，查看当前目录下的文件

    输入上面可以看到这个目录下的文件，Java 压缩包已经在里面了，现在就是要解压出来


    输入命令

    tar -zxvf jdk-8u201-linux-x64.tar.gz

1.3 配置环境变量

  1. 编辑全局变量：

    vim /etc/profile

  2. vim 是文本编辑命令，即编辑全局变量文件 profile

  3. 进入文本编辑状态后，shift+G 让光标移动到最后一行，再按下 i 键，进入插入状态
    ；

  4. 在文本的最后一行粘贴一下内容：

    #java enviroment
export JAVA_HOME=/root/jdk1.8.0_201
export CLASSPATH=.:${JAVA_HOME}/jre/lib/rt.jar:${JAVA_HOME}/lib/dt.jar:${JAVA_HOME}/lib/tools.jar
export PATH=$PATH:${JAVA_HOME}/bin

    退出并保存：（记得按 esc 键）

    :wq!

1.4 查看是否配置成功

  • 输入以下命令 
    source /etc/profile  执行代码让系统读取新的环境变量
java -version        执行代码可以查看JDK是否成功配置

2. MySQL的配置

1. 安装前准备


1.1 检查环境

  • 安装前还是先看下当前环境是否已经安装了 MySQL
    1. 检查是否已经安装了 MySQL

    rpm -qa | grep mysql

  • 如果已存在，则执行删除命令 后边为 Mysql 目录

    rpm -e --nodeps mysql-xxxx

  • 查询所有 Mysql 对应的文件夹

    whereis mysqlm
find / -name mysql

  • 删除相关目录或文件

    rm -rf /usr/bin/mysql /usr/include/mysql /data/mysql /data/mysql/mysql

  • 验证是否删除完毕

    whereis mysqlm
find / -name mysql

  • 检查 mysql 用户组和用户是否存在，如果没有，则创建

    cat /etc/group | grep mysql
cat /etc/passwd |grep mysql
groupadd mysql
useradd -r -g mysql mysql

2.2 安装MySQL

  • 从官网下载是用于 Linux 的 Mysql 安装包

  • 在你的上传目录下找到 Mysql 安装包：

    mysql-5.7.24-linux-glibc2.12-x86_64.tar.gz

  • 执行解压命令：

    tar xzvf mysql-5.7.24-linux-glibc2.12-x86_64.tar.gz[

  • 移动并重命名

    mv mysql-5.7.26-linux-glibc2.12-x86_64 /usr/local/mysql

  • 创建 MySQL 用户和组并修改权限

    groupadd mysql
useradd -r -g mysql mysql
mkdir -p  /data/mysql              #创建目录
chown mysql:mysql -R /data/mysql   #赋予权限

  • 创建配置文件 my.cnf

    vim /etc/my.cnf

  • my.cnf 中的内容

    [mysqld]
bind-address=0.0.0.0
port=3306
user=mysql
basedir=/usr/local/mysql
datadir=/data/mysql
socket=/tmp/mysql.sock
log-error=/data/mysql/mysql.err
pid-file=/data/mysql/mysql.pid
#character config
character_set_server=utf8mb4
symbolic-links=0
explicit_defaults_for_timestamp=true

  • 初始化数据库

    进入 mysql 的 bin 目录

    cd /usr/local/mysql/bin/
./mysqld --defaults-file=/etc/my.cnf --basedir=/usr/local/mysql/ --datadir=/data/mysql/ --user=mysql --initialize

  • 查看密码

    cat /data/mysql/mysql.err

  • 启动 MySQL

    service mysql start
ps -ef|grep mysql

  • 到这里 MySQL 已经安装成功了

    修改密码，刚才的密码是随机生成的，需要设置自己的密码

    ./mysql -u root -p #在MySQL的bin目录下执行,这是登录MySQL

    在执行下面的操作，重新登录

    SET PASSWORD = PASSWORD('123456');
ALTER USER 'root'@'localhost' PASSWORD EXPIRE NEVER;
FLUSH PRIVILEGES;

    但是这个时候远程是连不上的

    登录数据库后执行下面的命令

    use mysql               #访问mysql库
update user set host = '%' where user = 'root';   #使root能再任何host访问
FLUSH PRIVILEGES;       #刷新

结束

  • 如果不希望每次都到 bin 目录下使用 mysql 命令则执行以下命令：

    ln -s /usr/local/mysql/bin/mysql /usr/bin

  • 因为我在虚拟机上都已经配置好了，就没有再重新配置，有机会的话，就重新配然后记录一下个更新这篇文章
